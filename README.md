# Тестовое задание LSR — форма заявки на объект недвижимости

## Стек
- Битрикс CMS (Bitrix D7)
- PHP 8.2
- HighloadBlock (3 HL блока)
- Vanilla JS (fetch API)

## Структура файлов

```
index.php                        — страница с формой
local/tools/form-handler.php     — AJAX-обработчик (backend)
```

## Настройка HL блоков в админке Битрикса

Перейти: **Настройки → Highload-блоки → Добавить**

### 1. Buildings (дома)
- Имя: `Buildings`
- Таблица: `hl_buildings`

| Поле      | Тип         | Описание       |
|-----------|-------------|----------------|
| UF_NAME   | Строка (100) | Название дома  |

### 2. Apartments (квартиры)
- Имя: `Apartments`
- Таблица: `hl_apartments`

| Поле            | Тип          | Описание                      |
|-----------------|--------------|-------------------------------|
| UF_BUILDING_ID  | Целое число  | ID дома (из Buildings)        |
| UF_NUMBER       | Строка (50)  | Номер квартиры                |
| UF_STATUS       | Строка (20)  | Статус: `FREE` / `RESERVED` / `SOLD` |

### 3. Applications (заявки)
- Имя: `Applications`
- Таблица: `hl_applications`

| Поле            | Тип           | Описание                   |
|-----------------|---------------|----------------------------|
| UF_NAME         | Строка (100)  | Имя заявителя              |
| UF_EMAIL        | Строка (100)  | Email                      |
| UF_PHONE        | Строка (50)   | Телефон                    |
| UF_APARTMENT_ID | Целое число   | ID квартиры (из Apartments) |
| UF_CREATED_AT   | Дата и время  | Дата подачи заявки         |

## Установка

1. Скопировать файлы в корень сайта Битрикс:
```bash
cp index.php /var/www/site/
cp -r local/tools /var/www/site/local/
```

2. Создать HL блоки по схеме выше.

3. Заполнить тестовыми данными (дома и квартиры) через интерфейс HL блоков в админке.

4. Открыть сайт — форма готова к использованию.

## Просмотр заявок в админке

Перейти: **Настройки → Highload-блоки → Applications → Элементы**

## Логика

- Форма динамически подгружает квартиры при выборе дома (только `FREE` и `RESERVED`)
- Квартиры со статусом `SOLD` не отображаются
- При отправке проверяется:
  - Валидность email и телефона
  - Дубль email или телефона **на тот же объект** → ошибка
  - Если квартира `SOLD` в момент отправки → «Выберите другой объект недвижимости»
  - CSRF-защита через `bitrix_sessid_post()`
- При успехе возвращается `{"status": "ok"}`

## AI-инструменты

При разработке использовался GitHub Copilot (Claude Sonnet 4.6).

Команды и настройки:
- Сгенерировать структуру файлов и бизнес-логику обработчика
- Настройка сервера: nginx/apache vhost, PHP 8.2 (remi), `short_open_tag`, `session.save_path`, `opcache.revalidate_freq`
- Рефакторинг: замена `$_POST`/`$_SERVER` на Битрикс D7 `HttpRequest`
